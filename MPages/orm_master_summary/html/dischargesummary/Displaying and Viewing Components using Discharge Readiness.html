MPages Help Pages : Displaying and Viewing Components using Discharge Readiness
This page last changed on Aug 09, 2011 by JP3026.

To display clinical information from the Discharge Readiness MPages view, follow the steps below. There are a couple of ways of displaying and viewing information. You can follow one or more of the options described below to display the appropriate data.

Option One:

  1. Navigate to the Patient list you want. See the topic Displaying the Patient List using Discharge Readiness and Searching the Patient List using Discharge Readiness for more information.
  2. Once the patient list you want is displayed, each patient row becomes a click-able target. Click a row to expand it. The system displays the Clinical Summary Dashboard view. The Clinical Summary Dashboard view contains a header of patient information at the top with Patient Name, MRN, FIN, Attending Physician, DOB, etc. Below the patient information header are the applicable components that contain clinical data for Problems, Diagnoses, Allergies, Home Medications, Diagnostics, Documents, Vital Signs, Labs, etc.
  3. If a "+" plus sign is displayed to the right of the component title, select it to expand the component. Below are some examples describing the expanded component data.
    1. Diagnosis component - If a search box is displayed below the component header, enter a few characters of a nomenclature search term to display a matching list. Select the result you want from the drop down list, and the component data is displayed below it.
      Release Considerations
      Beginning with MPages Content Package 53272 ( MPages Component Library 2.5 (August 2011)) and MPages Bedrock Configuration Content Package 53281 (MPages: Discharge Readiness Configuration 2.0 (August 2011)) and its replacement, the Quick Add Search functionality described above was added. If you plan to use this functionality, make sure server 396 is running and the correct preferences are configured in Preference Maintenance Tool (PrefMaint.exe). For details on the functionality and the Preference Maintenance Tool configuration, see the Problems and Diagnosis design component pages.
    2. Documents component - You can update a PowerNote and view documented results using this component.
      1. Click a results row that contains a blue underlined link, which means there are documented results. The Document Viewer window is displayed.
      2. To update the PowerNote on the patient's chart, click the fourth icon over from the left on the toolbar in the Document Viewer window. The system launches the Open Note window.
      3. Select either Current Encounter or All Encounters on the Existing tab. Select the note you want from the list and click Make Correction. The system displays the Open Note editor where you can modify the note.
        Note
        The Open Note window contains tabs for Encounter Pathway, Existing, Precompleted, Catalog, Recent, and Favorites.
      4. Close the open windows to return to the MPages view.
        Release Considerations
        Beginning with MPages Content Package XXXXX ( MPages Component Library 4.0 (July 2011)) and MPages Bedrock Configuration Content Package XXXXX, XXXXX, XXXXX, and XXXXX (MPages: XXXXX Summary Configuration 4.0 (July 2011)) and its replacement, the Documents Results viewer enhancement described above became available.

Option Two:

  1. Navigate to the MPage Summary view.
  2. If it is not already expanded (you see a minus sign next to the component title), expand the component as described in Option One above.
  3. Position your mouse over a result row or column to display a pop-up window containing additional details. See the MPage components described below.
    Note
    These are just a few of those offered.
    1. Diagnosis component - Displays the diagnosis code and diagnosis description for the current or selected encounter.
    2. Patient Information component - Contains the patient Reason for Visit, Primary Physician, Attending Physician, Room and Bed, Admission Date, Anticipated Discharge Date, and other basic pieces of patient information.
    3. Outstanding Orders component - Displays the Order Name, Order Date and Time, Order Catalog Type, and Order Status for the current or selected encounter.
