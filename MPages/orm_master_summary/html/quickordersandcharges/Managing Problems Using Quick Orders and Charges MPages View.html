MPages Help Pages : Managing Problems Using Quick Orders and Charges MPages View
This page last changed on Jan 03, 2012 by HE3726.

Release Considerations
Beginning with MPagesContent Package 55241 (MPages (US): Quick Orders and Charges (QOC) MPages View (November 2011)) and its replacement, the quick orders functionality described in this Reference Page became available. This particular package is currently available to testing partners only.

Managing problems and diagnoses is frequently a part of the ordering workflow. Therefore, the ability to add new problems and diagnoses to a patient's chart is available using the Quick Orders and Charges MPagesview. To complete these actions, follow the steps below: (ensure the Consolidated Problems component is expanded by clicking the icon in the upper right corner of the Consolidated Problems component)

  1. Adding a new diagnosis, problem or past medical history.
    1. Below the component heading, the label of "Add new as" provides several options for adding a new problem, diagnosis and/or past medical history. These options include: This Visit, This Visit and Active, Active, Historical.
    2. Within the search field, begin typing the desired item.  The search will display up to 10 of the closest matches based on the text entered.  When a nomenclature term is selected from the search and "This Visit" has been selected from the drop-down list, the condition is added to the "This Visit" section, and it is added as a diagnosis using the target vocabulary defined in Bedrock.
    3. When a nomenclature term is selected from the search and This Visit and Active are selected from the drop-down list, the condition is
      added to the This Visit section as a diagnosis and to the Active section as an active problem using the appropriate target vocabularies defined in Bedrock.
    4. When a nomenclature term is selected from the search and Active was selected from the drop-down list, the condition is added to the
      Active section as an active problem using the target vocabulary defined in Bedrock.
    5. When a nomenclature term is selected from the search and Historical was selected from the drop-down list, the condition is added to
      the Historical section as a resolved problem.
    6. Items can also be dragged and dropped from one section to another (e.g. drag a problem into the "this visit" section to make it a diagnosis).  When this is done, the system will apply the appropriate vocabularies as defined in Bedrock.
  2. Performing the actions of Resolve, Inactive and Cancel
    1. A drop-down list within the component displays the following options: Resolve, Inactivate, and Cancel.
    2. When a condition in any of the subsections is selected, you are given an indication that the focus has been placed on the condition.
  3. When focus is placed on a problem in the Active section and Resolved is selected from the drop-down list, the problem status is updated to
    Resolved and the problem is no longer displayed in the Active section, and it is displayed in the Historical section.
  4. When focus is placed on a problem in the Active section and Inactive is selected from the drop-down list, the problem status is updated to
    Inactive and the problem is no longer displayed in the Active section, and it is displayed in the Historical section.
  5. When focus is placed on a problem in the Active section and Canceled is selected from the drop-down list, the problem status is updated to
    Canceled and the problem is no longer displayed.
  6. When focus is placed on a condition in any of the three sections, the actions of Resolve, Inactivate, and Cancel are available with the following exceptions:
    1. If the condition only consists of a diagnosis, the actions of Resolve, Inactivate, and Cancel are not available.
    2. If the condition includes problems, and the most recent problem is in a Resolved status, the Resolve action is not available.
    3. If the condition includes problems, and the most recent problem is in an Inactive status, the Inactivate action is not available.
  7. When the Resolve, Inactivate, or Cancel action is taken on a condition that includes a diagnosis, no action is taken on the diagnosis.
  8. When the Resolve action is taken on a condition, the status of the most recent problem changes to Resolved.
  9. When the Inactivate action is taken on a condition, the status of the most recent problem is updated to Inactive.
  10. When the Cancel action is taken on a condition, the status of the most recent problem is updated to Canceled.
  11. The Resolve, Inactivate, and Cancel actions can be taken on conditions that are free-text problems with the following exceptions: a problem in a Resolved status does not have the Resolve action available, and a problem in an Inactive status does not have the Inactivate action available.
  12. Multiple conditions can be selected within multiple sections, and they can use the Resolve, Inactivate, or Cancel action when each selected condition satisfies the above conditions.
