Overview

The Diagnoses component displays all active diagnoses for this encounter for the patient sorted alphabetically. If there is a codified nomenclature, the code is displayed in parenthesis next to the name. Clicking on the section header takes you to the Problems and Diagnoses List in the chart.

To find additional information such as the diagnosis dates or comments, you can hover over the fields.

When adding a diagnosis, the search capabilities are enhanced to suggest search results and refine the choices as you type each character. Once a term has been selected, the term displays in the Diagnosis view and is automatically saved.

A numeric value can be entered in the search box and the closest matches among the terminology used in the search are displayed.

Note
When you search for an item and a duplicate is found, you will not be allowed to add the duplicate item.
Component Details

Heading

  • Diagnoses (n active)
    n = total number of active diagnoses
  • Component heading links to appropriate component in the chart

Content

  • Display diagnosis name
  • Display associated code in parenthesis

Data Format

  • Look Back Range: current, selected encounter
  • Status: display only active diagnoses
  • Sort Order: Display diagnoses in alphabetical order
  • Truncate / Wrap: text wraps if it exceeds the column width
  • Scrolling: Client-defined

Hover

  • Diagnosis: Followed by the name of the diagnosis.
  • Annotated Display: Followed by the annotated display name of the diagnosis.
  • Diagnosis Date: (MM/DD/YYYY)
  • Comments: Followed by all associated comments for the diagnosis (first line: documented date - documented by name / second line: comment text).
