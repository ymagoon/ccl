univnm. ext. Statusbar

A status bar you can incorporate into your application.  Has elements of a profiler in it, and internally uses and augments the standardized profiler.

Examples

Ext.define("MPAGE.store.People", {
    extend: "Ext.data.Store",
    constructor: function(config) {
        this.callParent([config]);
        this.on({
            beforeload: function(store) {
                store.on({
                    load: $profiler.status("Loading People"),
                    single: true
                });
            }
        });
    }
});

Profiling

Any status update is automatically profiled in the debug window.  The call to $profiler.status() returns a function to mark that action complete.  It’s equivalent to:

$profiler.begin("test")

Statusbar will provide timing information after the status text.  That is, $profiler.status(“Loading”) will produce text: “Loading (running for 1 second)”, “Loading (running for 2 seconds)”, etc.  The timing information can be surpressed by setting a showTime: false in the object configuration:

{
    xtype: "panel",
    title: "Main Panel",
    dockedItems: [
        {
            xtype: "asyncstatusbar",
            showTime: false
        }
    ]
}

Additional Buttons

You can attach additional buttons (or other objects) to the status bar.

{
    xtype: "asyncstatusbar",
    buttons: [
        {
            xtype: "button",
            text: "Availability",
            menu: {
                items: [
                    {
                        text: "Online"
                    },
                    {
                        text: "Offline"
                    }
                ]
            }
        }
    ]
}
