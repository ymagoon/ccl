The Allergies component displays all active (and proposed) allergies and the associated reaction for the lifetime of the patient.

Allergies are sorted by severity, and then alphabetically. The highest severity allergies will display in a bold, red font.

Hovering over an allergy will display the allergy name, severity, reactions, onset date, and any comments. Clicking on the section header will take you to the Allergy Profile in the chart, and clicking on the Add button will take you to the Allergy profile with the Add Allergy conversation already started.

I'll click Add, and when the add allergy window appears I want to add a morphine allergy. Using the catalog of common drug allergies, I select morphine, search for a reaction of hives, this is a severe reaction, information source is my patient, and we're not sure when it started. I'll add a comment to test the display. Now I'll return to the Ambulatory Summary by clicking in the Table of Contents, refresh my screen, and you can see my morphine allergy.
Component Details

Heading

  • Allergies(n active)
    n = total number of active allergies
  • Component heading will link to appropriate component in the chart
  • +Add button that will open the add conversation in the Allergy tab

Content

  • Display allergy name
  • Display associated reactions to the allergy

Data Format

  • Look Back Range: lifetime of the patient
  • Status: display active and proposed allergies
  • Sort Order: sorted by severity, and then alphabetically. Display allergies and associated reaction with anaphylaxis and severe severities in a bold, red font
  • Scrolling: client defined
  • Truncate / Wrap: text will wrap if it exceeds the column width

Hover

  • Allergy Name
  • Reactions
  • Severity
  • Status
  • Onset Date
  • Comments
