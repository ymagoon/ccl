The Medications component displays all active home medications and prescriptions for the lifetime of the patient.
Medications will display along with dose, route, and frequency, if documented.
Medications are sorted alphabetically, and if there are more than 10 active medications a scroll bar will appear.
Correction to the video - medications are grouped by prescription and home medication with prescriptions displayed first, and then sorted alphabetically within each group.

Positioning the mouse over a medication will display the medication name, ordered as mnemonic, order date, signature line, and who it was ordered by. Clicking the section header will take you to the Medication List in the chart, and clicking on Add will take you to the Medication List with the Add Order conversation already launched.

I'll click on Add, and when the order search window appears I want to add a prescription for Vitamin D. I'm in the Meds as Rx venue type already. Search for Vitamin D. Choose an order sentence. All my required fields are populated so I click sign. I refresh the order profile. I see my prescription in an ordered status, and now I'll return to the Ambulatory Summary by clicking in the Table of Contents. Click Refresh, and my Vitamin D prescription is displaying on the MPage.
Component Details

Heading

  • Medications (n Rx / n Hx)
    n = total number of active prescriptions or documented meds
  • Component heading will link to appropriate component in the chart
  • +Add will launch the add orders conversation in the Medication List

Content

  • Display label of Rx: or Hx: followed by the medication name
  • Display dose, route, frequency, and number of refills

Data Format

  • Look Back Range: lifetime of the patient
  • Status: display only active prescriptions and home medications
  • Truncate / Wrap: text will wrap if it exceeds the column width
  • Scrolling: client defined

Hover

  • Medication Name
  • Ordered as Mnemonic
  • Order Date
  • Sig Line (order details)
  • Ordered By
  • Order Type
